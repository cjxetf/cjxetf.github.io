# 传统编程 vs Vibe Coding

> **Vibe Coding**：通过自然语言、注释和上下文“营造编程氛围”，由 AI 助手（如 GitHub Copilot、Cursor）自动生成或补全代码。

| 维度 | 传统编程 | Vibe Coding |
|------|----------|-------------|
| **核心焦点** | **How（如何实现）**<br>手动编写每一行逻辑 | **What（想要什么）**<br>描述意图，让 AI 生成实现 |
| **主要工具** | IDE + 官方文档 + Stack Overflow | AI 编程助手（Copilot / Cursor / Windsurf）+ 自然语言 |
| **编码方式** | 逐行手写、调试、重构 | 写注释/函数签名 → AI 补全 → 审查/微调 |
| **认知负荷** | 高（需记忆语法、API、模式） | 低（聚焦业务逻辑，AI 处理样板） |
| **开发速度** | 中等（熟练者快） | **极快**（简单任务秒级生成） |
| **代码可控性** | **高**（完全由开发者掌控） | 中（依赖 AI 理解“vibe”，需人工审查） |
| **典型工作流** | 设计 → 编码 → 测试 → 调试 | 描述意图 → AI 生成 → 验证 → 迭代 |
| **适用任务** | - 核心算法<br>- 高安全模块<br>- 性能敏感代码 | - CRUD / DTO<br>- 原型/MVP<br>- 测试桩<br>- 跨语言迁移<br>- 文档即代码 |
| **学习重点** | 语言特性、设计模式、调试技巧 | **Prompt 工程**、意图表达、AI 输出批判性验证 |
| **风险** | 人为 bug、效率低 | **AI 幻觉**、安全漏洞、技术债、过度依赖 |
| **团队协作** | 代码即文档 | **注释/注解即 Prompt**，需更清晰的意图表达 |
| **调试体验** | 可预测，栈跟踪清晰 | 可能“黑盒”，需理解 AI 生成逻辑 |

## ✅ 适用建议

- **用传统编程当**：
     - 实现加密、支付、权限等**高风险逻辑**
     - 优化性能瓶颈
     - 教学/深入理解底层机制

- **用 Vibe Coding 当**：
     - 快速搭建原型
     - 生成重复性代码（如 API Controller、Entity Mapper）
     - 探索不熟悉的库或语言
     - 提升日常开发效率（配合严格 Code Review）

> 💡 **未来趋势**：  
> **不是“取代”，而是“增强”** —— 优秀开发者 = **强抽象能力 + 精准 vibe 表达 + 批判性验证**。
